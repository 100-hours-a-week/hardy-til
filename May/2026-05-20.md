SOLID
- 유지보수성 + 확장성 높이려고 만든 5가지 법칙
- S (SRP) - 단일 책임 원칙 -> 클래스를 수정해야하는 이유 단하나
  - 변경 영향이 적음
  - 가독성이 올라감
- O (OCP) - 개방 폐쇄 원칙 -> 확장은 열고 기존 코드 수정은 닫아라
- L (LSP) - 리스코프 치환 법칙 -> 자식 클래스는 언제나 부모 클래스를 대체할 수 있어야한다.
- I (ISP) - 인터페이스 분리 법칙 -> 내가 안쓰는 거대한 인터페이스보다, 쪼개진 인터페이스를 쓰자
- D (DIP) - 의존관계 역전 -> 구체적인 것에 의존하지 말고, 추상적인 것에 의존해라

JVM (면접 질문 단골)
- 바이트코드를 읽어서 기계어로 바꾸는 역할 담당
- 구성 
  - 클래스 로더
    - 프로그램 실행 필요한 .class 파일을 JVM에 가져오는 역할
    - 특징
      - 자바는 프로그램을 시작할 때 모든 파일을 한 번에 불러오지 않음
      - Loading : 클래스 파일을 JVM 메모리에 롣
      - Linking :  클래스 파일 사용을 위해 검증
        - Verifying : JVM 호환 확인
        - Preparing : 메모리 할당
        - Resolving : 클래스 상수 풀 내 모든 심볼링 레퍼런스를 다이렉트 레퍼런스로 변경
      - Initialization : 클래스 변수를 적절한 값으로 초기화
  - 실행 엔진
    - 배치된 바이트 코드를 읽어서 실행
    - 자주 쓰는 코드는 기계어로 미리 번역
    - GC
  - Runtime Data Areas
    - JVM의 메모리 영역
    - 구성
      - 메서드 영역
        - 바이트 코드 안에 명세되어 있는 클래스 정보들이 여기로 들어옴
        - 콘스탄트 풀, 필드/메서드 데이터, 메소드 코드, 생성자
        - 런타임 콘스탄트 풀
      - 힙
        - 데이터를 저장하기 위해 런타임 시 동적으로 할당
        - 모든 쓰레드가 공유
        - 레퍼런스 타입을 저장
      - 스택
        - 메서드를 실행할 때 사용하는 임시 저장 공간
        - 기본형 변수나 메서드 실행 정보 저장
        - 스레드마다 가지고 있음
      - PC 레지스터
      - 네이티브 메서드 스택
